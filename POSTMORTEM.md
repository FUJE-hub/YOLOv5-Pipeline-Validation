# 实验总结：从环境卡点到模型优化——一次完整的深度学习工程实践复盘

> 本文档是对“基于YOLOv5的火焰检测模型迭代优化实验”的完整技术复盘。重点不在于复述实验流程，而在于**还原真实的问题解决路径、决策逻辑与工程思维**，记录我从“环境配置”到“模型跑通”的全过程思考。

## 一、核心结论：量化结果与验证闭环
- **量化提升**：通过“增加训练轮数 + 启用早停”策略，模型召回率（Recall）从 **0.37 提升至 0.45（+21.6%）**，mAP@0.5 从 **~0.31 提升至 ~0.43（+38.7%）**。
- **关键验证**：实验完整走通了 **“数据准备 → 基线训练 → 问题定位 → 策略调整 → 效果验证”** 的迭代闭环，证明了“增加训练轮数”对缓解“高精度、低召回”问题的有效性。
- **工程产出**：本次实践产出了**结构化实验记录、可视化证据链、可复现的配置文档**，以及这份聚焦于**决策过程**的复盘总结。

## 二、核心决策复盘：从“追新踩坑”到“务实跑通”
### 1. 环境配置：在“版本新颖性”与“环境稳定性”间的权衡
- **问题**：初始安装时未严格核对版本兼容性，安装了较新的CUDA驱动，后发现与PyTorch官方稳定版不匹配，导致`torch.cuda.is_available()`返回`False`，GPU加速不可用。
- **尝试**：调研并尝试安装PyTorch的nightly（前瞻）构建以适配高版本驱动。
- **评估**：前瞻版虽能识别GPU，但依赖易冲突，且稳定性未经验证，在工程实验中引入了不确定性风险。
- **决策**：**果断回退至CUDA与PyTorch官方推荐的稳定版本组合**，明确“**环境稳定性 > 驱动版本新颖性**”的原则。
- **提炼**：深度学习环境配置**应始于严格的版本核对**，优先选择社区验证充分、文档齐全的稳定组合，避免事后补救。

### 2. 工具调试：当“完美脚本”阻碍“实验进度”时的务实转向
- **问题**：为了一键化环境诊断，用记事本编写Python脚本，因**输入法符号差异**（如中文括号、全角空格）导致路径解析报错，调试陷入细节。
- **尝试**：初步调试后问题依旧，脚本未能如期运行。
- **评估**：查询文档得知，YOLOv5s作为轻量模型，在100~300轮小规模实验中，CPU训练时间（约为GPU的3-5倍）仍在可接受范围内，且可**完全绕过CUDA兼容性问题**。
- **决策**：**放弃继续调试脚本**，直接在训练命令中追加 `--device cpu` 参数，强制使用CPU执行训练。
- **结果**：此举虽牺牲了训练速度，但**在半小时内解决了环境卡点**，使实验流程得以持续推进，并最终成功完成两轮训练。
- **提炼**：当工具调试陷入细节（符号、路径、依赖）时，应优先评估**是否有更直接、更可控的替代方案**。在实验初期，**“跑通闭环”的优先级应高于“工具完美”**。

### 3. 决策逻辑总结
| 关键节点 | 我的选择 | 决策依据 | 体现的工程思维 |
|:---|:---|:---|:---|
| **CUDA版本冲突** | 回退至稳定版 | 稳定性风险 > 版本收益 | **风险规避原则** |
| **脚本报错** | 改用CPU训练 | 实验进度 > 工具完美 | **务实优先原则** |
| **方案选择** | 接受CPU速度损失 | 时间可接受 + 绕过复杂问题 | **权衡取舍（Trade-off）** |

## 三、方法论提炼：从“操作步骤”到“思维模式”
1.  **问题定位**：不依赖直觉，通过**混淆矩阵（观察漏检）** 与**训练曲线（观察收敛）** 等量化工具定位“低召回”问题，明确优化方向。
2.  **策略选择**：针对“欠训练”假设，选择**增加训练轮数**这一低风险、高收益的调整项，并启用**早停机制**防止过拟合，形成完整优化回路。
3.  **评估标准**：坚持以**Recall和mAP@0.5为核心量化指标**，辅以混淆矩阵、检测效果图等可视化证据，确保评估客观全面。

## 四、工程实践建议（来自实战踩坑）
- **环境管理**：使用Conda创建独立环境，并通过 `conda install pytorch...` 命令安装，让包管理器自动处理CUDA与cuDNN依赖，最大程度避免环境冲突。
- **实验规范**：所有实验的**配置文件、训练日志、关键图表**均应按“基线/优化”分类存档于`ARCHIVE/`目录，形成可追溯的证据链。
- **协作流程**：采用 **“新分支 → Pull Request → 合并”** 的GitHub标准流程。本次实验的所有修改（包括本总结文档）均通过此流程提交，保证了主分支稳定与修改可追溯。

## 五、未来方向
- **数据层面**：针对小目标漏检，引入**困难样本增强**（如Mosaic、MixUp）。
- **模型层面**：尝试**YOLOv5m/l**等更大模型，探索精度与速度的平衡点。
- **部署验证**：将优化后的模型转换为**TensorRT/ONNX**格式，在边缘设备（如Jetson Nano）上测试实时性能，完成从实验到部署的最后一公里验证。

本次实验远不止一次“调参”。它是一次完整的**深度学习工程思维训练**——从环境配置的“踩坑”与“回退”，到工具调试时的“取舍”与“转向”，再到实验设计的“假设”与“验证”。
